KMS激活脚本KMS-VL-ALL Final汉化版

  • 激活
  • 2018-07-29

前言

  KMS-VL-ALL是一款出自国外My Digital Life论坛大神制作的KMS激活脚本，可以在线/离线激活Windows、Office及VL版批量授权版产品，相比其它激活工具，这款批处理版KMS激活脚本具有体积小、代码开源/简洁、操作简单、无残留等特点，这脚本激活成功率总体来说还是很不错！


KMS-VL-ALL功能简介

  • 支持激活Visual Studio、Windows、Office及VL版批量授权版产品：

    • 🚡 Windows Vista/Win7/Win8/8.1/Win10 (v1709 RS3) 以及所有VL版
    • 🚡 Windows Server 2008/2008R2/2012/2012R2/2016 (v1607) 所有版本
    • 🚡 Windows XP SP3 或更高系统上的 Office 2010 系列
    • 🚡 Windows 7 或更高系统上的 Office 2013 系列
    • 🚡 Windows 7 SP1 或更高系统上的 Office 2016 系列
    • 🚡 Visual Studio 2013 Ultimate 旗舰版
    • 🚡 Visual Studio 2015 Enterprise 企业版
    • 🚡 Visual Studio 2017 Enterprise 企业版

  • 支持零售/MAK版的未激活版本：

    • 🚡 Windows 7/8/8.1/10 专业/企业版 零售版/MAK
    • 🚡 Windows Server 2008/2008 R2/2012/2012R2/2016 零售版/MAK
    • 🚡 Windows Office 2010/2013/2016 MAK Windows Office 2016 C2R

注：零售/OEM/MAK已经永久激活的不影响。并将通知期/OOBE 宽限期的 Windows 转换为 VL （如果它们受支持），并激活。

使用FAQ

💘 如何使用这个脚本激活产品？
  解压到英文路径，并以管理员身份运行KMS-VL-ALL.cmd

💘 如何将Office 2016 C2R 转换VL版本？
  7.0以前版本，以管理员身份运行Convert-Office2016C2R.cmd
  7.0以后版本，以管理员身份运行Convert-Office.cmd

💘 脚本提示“未检测到 Office 2013 及以上版本的 VL 产品；零售版本需要先转换为 VL 版本。” 此类问题属于可能没有正确安装 office 产品所致，或者你安装的是零售的版本。对于没有正确安装的用户，应重新安装对应的版本；对于零售版本的用户，你可以更换版本，也可以执行授权转换。现在的 6.9 已经配备了相应的授权转换脚本，也可以用单独版本的授权转换脚本执行授权转换。

💘 脚本的激活过程中有提示“无法检测到支持 KMS 客户端的 Windows 版本……” 这个在已激活的系统中常见。对于 Windows 10，但凡你是数字激活的系统，都会这样提示。如果你使用的是盗版 Windows 7，8，8.1，同样会出现此提示。但你不用担心，这不会因此丢掉已有系统的激活，脚本将会自动跳过激活 Windows 的步骤。

💘 脚本提示“未检测到 Office 2010 VL 产品；零售版本需要先转换为批量授权。” 这个原因很明显，一种原因是你有可能没有正确安装 Office，一种原因是你可能装的是零售版本。这和 Office 2013/16 有很大区别。对于 Office 2010 及以下版本，你可以将已安装的零售版卸载更换为 vl 版本，或者通过其他的工具激活。虽然这部分用户现在不是特别的多，但这个问题确实需要注意。
